Spatial Audio Volume

Two apps. One head. Move closer to one — it gets louder. The other quiets down.

Windows · Tauri 2 · Rust

What it does

  • Lists every audio session on your default render endpoint — same list as Settings → Sound → Volume Mixer.
  • Two slots, left and right. A draggable "you" avatar between them sets both volumes from a single position.
  • Sessions identify by PID, so two windows of the same app don't move together.
  • Drives WASAPI directly. No virtual devices, no rerouting.

Run it

Prerequisites: Rust toolchain and cargo-tauri. Node is not required — the frontend is plain static files.

cargo tauri dev

For a release build:

cargo tauri build

The bundled installer lands in src-tauri/target/release/bundle/.

Under the hood

The audio backend talks to WASAPI's IAudioSessionManager2 and ISimpleAudioVolume via the windows crate. The avatar's axis position maps to two complementary gains: at -1 the left slot is at 100% and the right at 0%; at +1 it's the opposite; at 0 both are balanced.

Frontend is vanilla HTML/CSS/JS — no bundler, no framework. The hand-drawn feel comes from a single SVG field with sketchy strokes and the Caveat font.
